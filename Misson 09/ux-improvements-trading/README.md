## 🔍 핵심 인사이트 도출

Mission 08에서 수집한 행동 데이터를 4개의 핵심 인사이트로 정리했습니다.

### Insight 1 ★ Critical — 비수탁성이 사용자에게 전달되지 않음

**정량 근거:**
- T6 정답률 **33% (1/3)** — 강민석(2년 경력), 김주형(3개월) 모두 실패
- W2 대시보드 누적 멈춤 시간 **45초**
- 평균 시선 이동 4회, 클릭 시도 0회 (김주형)

**정성 근거 (참가자 발화):**
- 강민석: *"여기 플랫폼 내부에요"* → 정답과 정반대 인지
- 김주형: *"앱에 보관되는 거 아닌가요?"*
- 정답 공개 후 강민석: *"그게 진짜 차별점인데 너무 숨겨져 있어요"*

**해석:**
2년 경력 사용자조차 핵심 가치를 놓친다는 것은 UI 문제. 랜딩에서 본 정보가 대시보드로 오면서 사라지기 때문.

---

### Insight 2 — W3 GPT 전략 화면의 학습 비용이 가장 높음

**정량 근거:**
- W3 누적 멈춤 시간 **52초** (6개 화면 중 최대)
- T3 완료율 **66%** (김주형 시간 초과 실패)
- 오조작 **총 5회** (P2 3회, P3 3회)

**정성 근거:**
- 노경민: *"이거 누르면 뭐가 뜨는 거예요?"* (블록 클릭 후 반응 없음)
- 강민석: *"어떻게 직접 수정하지? 클릭하면 모달 같은 게 떠야 하는 거 아닌가요?"*
- 노경민: *"응답 기다리는 게 좀 답답하네요"* (GPT 응답 대기 5초)

**해석:**
No-Code 편집기의 **인터랙션 어포던스 부재**가 근본 원인. 사용자는 블록이 클릭 가능한지, 뭐가 일어날지 예측 불가.

---

### Insight 3 — W6 예외 화면은 놀랍도록 성공적 (유지 필요)

**정량 근거:**
- T5 완료율 **100%**
- 평균 이해 시간 **11초** (6개 태스크 중 최단)
- 누적 멈춤 시간 **0초**
- 오조작 **0회**

**정성 근거:**
- 강민석: *"이건 진짜 자동매매다운 모습인데요? 수업 중에 이거 떴으면 안심하고 수업 계속 들을 수 있겠어요"* ★ (페르소나 시나리오 정확 매칭)
- 김주형(초보): *"RL 엔진이 뭔지는 모르겠지만 뭔가 알아서 해주는 것 같아요"* (개념 모르지만 UI만으로 정확 인지)

**해석:**
**4-Step 진행 트래커의 시각적 명확성**이 사용자 층위 무관 통했음. 이 패턴을 다른 화면에도 확장 가능.

**개선안: 유지 + 확장** (변경 아님)

---

### Insight 4 — 백테스팅 KPI 용어 장벽 (초보자)

**정량 근거:**
- W4 김주형 멈춤 시간 **22초** (P1은 0초, P2는 4초)
- 김주형은 클릭 없이 화면만 응시

**정성 근거:**
- 김주형(초보): *"MDD가 뭐예요? 마이너스 8.2%면 안 좋은 거 같은데?"*
- 김주형: *"Sharpe Ratio 1.84... 이게 좋은 수치인가요?"*
- 강민석: *"작년 BTC 40% 올랐는데 이 24%가 잘하는 건가요?"* (벤치마크 부재)

**해석:**
**절대값만으로는 성과 판단 불가**. 컨텍스트(용어 설명 + 비교 기준)가 필요.

## 🔄 Before / After 개선안

### 개선안 1: 비수탁성 인지 강화 (W2 대시보드) — P0

![Before/After 1](./assets/before-after-01.png)

#### 개선 항목

| # | 위치 | 변경 내용 |
|---|---|---|
| 1️⃣ | 대시보드 상단 | **영구 신뢰 배너 추가**: *"🔒 내 자산은 거래소에 그대로 보관됩니다 (비수탁형)"* |
| 2️⃣ | 자산 위젯 하단 | **"업비트에 보관 중 (비수탁)" 라벨 명시** |

#### Rationale (근거)

| 근거 유형 | 데이터 |
|---|---|
| 정량 | T6 정답률 33% → 목표 90%+ |
| 정량 | 자산 위치 탐색 시간 45초 → 목표 5초 |
| 정성 | 강민석 발화 "너무 숨겨져 있어요" |
| 심리 | 반복 노출을 통한 인지 강화 (2곳에서 노출) |

**왜 이 방식인가?** — 예뻐서가 아니라, **인지 반복 이론**에 근거함. 사용자가 한 번 정보를 못 봤을 때 다른 곳에서 다시 볼 수 있도록 **redundancy(중복 노출)**를 확보. 단, 시각적 노이즈 최소화를 위해 무채색 배너 사용.

### 개선안 2: GPT 전략 화면 어포던스 강화 (W3) — P0

![Before/After 2](./assets/before-after-02.png)

#### 개선 항목

| # | 위치 | 변경 내용 |
|---|---|---|
| 1️⃣ | 채팅 응답 대기 영역 | **로딩 스피너 + 상태 텍스트**: *"전략을 생성하고 있어요... 보통 5초 정도 걸립니다"* |
| 2️⃣ | 채팅 입력창 우측 | **글자수 카운터** (0/500) |
| 3️⃣ | 각 블록 우측 | **✏️ 편집 아이콘 + hover 시 활성 상태** |
| 4️⃣ | 블록 클릭 시 | **편집 모달** (자본 비율 슬라이더/입력) |

#### Rationale (근거)

| 근거 유형 | 데이터 |
|---|---|
| 정량 | W3 멈춤 52초 → 목표 20초 이하 |
| 정량 | T3 완료율 66% → 100% |
| 정량 | 오조작 5회 → 0회 |
| 정성 | 노경민 "이거 누르면 뭐가 뜨는 거예요?" |
| 정성 | 강민석 "모달 같은 게 떠야 하는 거 아닌가요?" |
| UX 이론 | Nielsen 휴리스틱 #1 (System Status Visibility) |

**왜 이 방식인가?** — 로딩 스피너는 예뻐서가 아니라, 사용자가 시스템이 응답하고 있음을 **인지**해야 이탈하지 않기 때문. 편집 아이콘도 장식이 아니라 **클릭 가능성을 시각적으로 신호**하는 최소 어포던스.

### 개선안 3: 백테스팅 컨텍스트 제공 (W4) — P1

![Before/After 3](./assets/before-after-03.png)

#### 개선 항목

| # | 위치 | 변경 내용 |
|---|---|---|
| 1️⃣ | 각 KPI 카드 하단 | **컨텍스트 문구 추가** ("시장 대비 +12.3%p", "평균 이상") |
| 2️⃣ | MDD/Sharpe 카드 | **ⓘ 아이콘 + hover 툴팁** (용어 설명) |
| 3️⃣ | Equity Curve | **BTC 단순보유 벤치마크 라인 추가** (점선) |

#### Rationale (근거)

| 근거 유형 | 데이터 |
|---|---|
| 정량 | W4 초보 이해 시간 38초 → 목표 15초 |
| 정성 | 김주형 "MDD가 뭐예요?" |
| 정성 | 강민석 "작년 BTC 40% 올랐는데 이거 잘하는 건가요?" |
| UX 이론 | Recognition over Recall (재인식 > 기억) |

**왜 이 방식인가?** — 툴팁이 예뻐서가 아니라, 초보 사용자가 **금융 용어를 몰라도 배포 결정을 할 수 있어야** 하기 때문. 벤치마크 라인은 **절대 성과 → 상대 성과**로 인지 프레임 전환. "이 정도면 좋은 건가?"라는 근원적 질문에 답을 줌.

### 우선순위 결정 근거

> **"Sprint 1만 완료해도 전체 임팩트의 65% 확보"**

**Impact × Effort 매트릭스 논리:**
1. **Quick Wins 우선** — 노력 대비 임팩트가 큰 개선을 먼저 (I1~I4)
2. **가설 실패 항목 우선** — H2(비수탁 인지)를 실패했으므로 관련 개선을 P0로 승격
3. **행동 데이터 빈도 순** — 오조작 5회, 멈춤 52초처럼 관찰 빈도가 높은 문제부터
4. **파워유저 개선은 후순위** — Cmd+Enter는 3명 중 1명만 시도

- ~~"대시보드가 좀 밋밋해 보이니 색깔을 추가하자"~~

- ~~"세련되게 아이콘을 추가하자"~~

| 개선안 | 미학적 판단 X | 데이터 판단 O |
|---|---|---|
| I1 비수탁 배너 | "안전해 보이는 아이콘" | *T6 정답률 33% → 목표 90%* |
| I3 로딩 스피너 | "예쁜 애니메이션" | *W3 대기 시간 5초 · P1 발화 "답답함"* |

| I6 벤치마크 라인 | "차트가 더 풍성해 보임" | *강민석 "잘하는 건가요?" 발화* |

**모든 개선안은 정량 데이터 + 정성 발화의 조합으로 정당화됨.**
