# 🤖 OpenAI 연동형 자동매매 시스템 — IA (정보 구조)

> **작성자: 박서연**
> Mission 03 / Information Architecture
> Last updated: 2025-05

GPT와 강화학습 기반으로 코딩 없이 나만의 투자 전략을 만들고,
여러 거래소에서 자동으로 매매해주는 AI 트레이딩 플랫폼의 정보 구조입니다.

## 🗂 전체 IA 구조 (Tree)

```
🏠 OpenAI 자동매매 시스템 (Root)
│
├── 1. 🚪 인증 & 온보딩  [Public]
│   ├── 1.1 랜딩 페이지
│   │   ├── 1.1.1 서비스 소개 (Hero)
│   │   └── 1.1.2 요금제 안내
│   ├── 1.2 회원가입
│   │   ├── 1.2.1 이메일 가입 폼
│   │   └── 1.2.2 이메일 인증
│   ├── 1.3 로그인 / 비밀번호 찾기
│   └── 1.4 온보딩 튜토리얼 (4-step)
│
├── 2. 📊 대시보드  [Main Hub]
│   ├── 2.1 통합 대시보드
│   │   ├── 2.1.1 자산 현황 위젯
│   │   ├── 2.1.2 운영중 전략 요약
│   │   └── 2.1.3 실시간 알림 피드
│   └── 2.2 빈 상태 (전략 없음 안내) ⚠️
│
├── 3. 🔗 거래소 연동  [API Vault]
│   ├── 3.1 연동된 거래소 목록
│   ├── 3.2 신규 거래소 추가
│   │   ├── 3.2.1 거래소 선택 (업비트/바이낸스/Bybit)
│   │   ├── 3.2.2 API 키 입력
│   │   └── 3.2.3 권한 검증 & 완료
│   └── 3.3 키 관리 (재발급/삭제)
│
├── 4. 🧠 전략 스튜디오  [Core Feature]
│   ├── 4.1 내 전략 라이브러리
│   ├── 4.2 GPT 전략 생성
│   │   ├── 4.2.1 자연어 입력 (채팅)
│   │   ├── 4.2.2 GPT 추천 결과 보기
│   │   └── 4.2.3 No-Code 블록 편집기
│   ├── 4.3 백테스팅
│   │   ├── 4.3.1 기간/종목 설정
│   │   └── 4.3.2 결과 리포트
│   ├── 4.4 전략 배포 (실거래 ON)
│   └── 4.5 전략 마켓플레이스 (커뮤니티 공유)
│
├── 5. 📈 거래 & 리포트  [Analytics]
│   ├── 5.1 실시간 거래 내역
│   ├── 5.2 수익률 분석
│   │   ├── 5.2.1 일/주/월별 차트
│   │   └── 5.2.2 전략별 성과 비교
│   ├── 5.3 RL 엔진 모니터 (강화학습 상태)
│   └── 5.4 세금 리포트 (CSV 내보내기)
│
├── 6. ⚙️ 설정  [Account]
│   ├── 6.1 프로필 관리
│   ├── 6.2 알림 설정 (푸시/이메일/카톡)
│   ├── 6.3 보안 (2FA, 비밀번호)
│   └── 6.4 결제 / 구독 관리
│
├── 7. 💬 지원  [Help]
│   ├── 7.1 FAQ
│   ├── 7.2 1:1 문의
│   └── 7.3 공지사항
│
└── ⚠️ 예외 화면  [System States]
    ├── E.1 404 Not Found
    ├── E.2 500 Server Error
    ├── E.3 API 연동 실패 (Unhappy Path)
    └── E.4 네트워크 오프라인
```

## 🧭 그룹핑 논리 (사용자 동선 기반)

| 그룹 | 사용 빈도 | 사용자 멘탈 모델 | 동선상 위치 |
|---|---|---|---|
| 1. 인증 & 온보딩 | 1회성 | "시작하기" | 외부 → 내부 진입점 |
| 2. 대시보드 | **매일** | "지금 상태 확인" | 메인 허브 (홈) |
| 3. 거래소 연동 | 초기 1회 + 가끔 | "내 자산 연결" | 설정 직전 단계 |
| 4. 전략 스튜디오 | **자주** | "전략 만들기" | 핵심 작업 공간 |
| 5. 거래 & 리포트 | **매일** | "결과 보기" | 대시보드 심화 |
| 6. 설정 | 가끔 | "내 정보 관리" | 우측 상단 진입 |
| 7. 지원 | 문제 발생 시 | "도움 필요" | 푸터/우하단 |

### 📐 동선 설계 원칙
1. **3-Click Rule** — 핵심 기능(전략 생성, 거래 내역)은 홈에서 3클릭 이내 접근
2. **빈도순 배치** — 좌측 사이드바 상단부터: 대시보드 → 전략 → 거래 → 거래소 → 설정
3. **위험 격리** — API 키 관리, 결제는 2FA 재인증 후 접근
4. **실패 가시성** — 예외 화면(E.3)은 단순 에러가 아닌 시스템의 자동 대응 과정을 보여줌
