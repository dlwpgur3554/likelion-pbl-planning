## 📐 명세 작성 규칙

### Trigger / Action 표기 원칙
- **Trigger**: 사용자 또는 시스템의 구체적 행위 (예: "클릭", "포커스 이탈", "10초 경과")
- **Action**: 그 결과로 발생하는 시스템 동작 (예: "POST /api/strategies 호출", "토스트 메시지 표시")
- 모호한 표현 (예: "처리한다", "보여준다") 사용 금지

### 예외 표기 카테고리
- 🔴 **차단성**: 액션 자체를 막음 (필수입력 미충족 등)
- 🟡 **경고성**: 진행은 가능하나 사용자에게 알림 (글자수 초과 등)
- ⚫ **시스템성**: 네트워크/API 오류 등 시스템 레벨 예외

### 우선순위
- **P0** — MVP 필수
- **P1** — 1차 출시 권장
- **P2** — 차후 추가
