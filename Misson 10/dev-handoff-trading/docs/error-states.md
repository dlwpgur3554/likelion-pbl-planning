## 🔴 1. Error States (에러 화면)

### E-404 · 페이지 없음

| 항목 | 값 |
|---|---|
| **트리거** | 미매칭 경로 |
| **HTTP Code** | 404 |
| **디자인** | 무채색, 미니멀 |

**UI 구조:**
```
[LOGO]

404
찾으시는 페이지가 없습니다

주소가 잘못되었거나 페이지가 이동/삭제되었을 수 있습니다.

[대시보드로 →]  [이전 페이지로]
```

**동작:**
- 자동 로깅: `analytics.track('404', { path: window.location.pathname })`
- 3초 후 대시보드 자동 이동 옵션 (사용자 설정)

---

### E-500 · 서버 오류

| 항목 | 값 |
|---|---|
| **트리거** | 서버 5xx 응답 |
| **HTTP Code** | 500 |
| **자동 복구** | 3회 재시도 (백오프: 1s / 3s / 9s) |

**UI 구조:**
```
[LOGO]

⚠️
일시적인 서버 오류가 발생했습니다

문제가 지속되면 고객센터로 문의해주세요.
Error ID: err_abc123 (문의 시 알려주세요)

[다시 시도]  [고객센터 문의]  [대시보드로 →]
```

**동작:**
- Sentry 자동 전송 (스택 트레이스 + 사용자 컨텍스트)
- 재시도 3회 실패 시 사용자 액션 유도
- Error ID 자동 생성해 클립보드 복사 가능

---

### E-API · API 오류 예외 화면 ★ (차별화)

| 항목 | 값 |
|---|---|
| **트리거** | 거래소 API 호출 실패 |
| **자동 폴링** | 5초 간격 |
| **UX 테스트 결과** | 100% 이해 성공, 평균 11초 |

**UI 구조 (4-Step 트래커):**
```
🔗 거래소  ⚠️  ← 사이드바 마커

[⚠️ 업비트 API 일시 오류 감지]
[시스템이 자동으로 재연결을 시도하고 있습니다. 사용자 개입은 필요하지 않습니다.]

자동 복구 진행 상태

┌──────────────────────────────────────┐
│ ✓ 오류 감지 완료                       │
│   12:34:21 · 오류 유형: 네트워크 타임아웃  │
├──────────────────────────────────────┤
│ ✓ 자동 재연결 성공                     │
│   12:34:36 · 2회 시도 끝에 복구 (15초 소요)│
├──────────────────────────────────────┤
│ ↻ RL 엔진 변동성 대응 중                │
│   포지션 자동 축소 (30% → 목표 15%)    │
├──────────────────────────────────────┤
│ ○ 매매 정상화 대기                     │
│   예상 완료: 약 1-2분 후                │
└──────────────────────────────────────┘

[RL 엔진 상태 보기] [이벤트 로그] [대시보드로 →]

⚠️ 자세한 분석은 5.3 RL 엔진 모니터에서 확인하세요
```

**동작:**
- 5초 폴링: `GET /api/system/recovery-status`
- 4-Step 완료 감지 시 자동으로 이전 화면 복귀
- 30초 응답 없음: "연결 복구 시도 중" 폴백
- 재연결 3회 실패: Critical 알림 (푸시 + 이메일 + 카톡)
- **AUTH 에러**: 별도 화면 분기 (사용자 액션 필요)
- **MAINTENANCE**: 예상 종료 시각 표시

**핵심 카피 원칙:**
> "사용자 개입 불필요"
> 페르소나 박서연이 "회의 중에 이거 떴으면 안심하고 회의 계속하겠어요"라고 느낄 수 있도록.

### E-OFFLINE · 오프라인 감지

| 항목 | 값 |
|---|---|
| **트리거** | `navigator.onLine === false` |
| **자동 복구** | 온라인 감지 즉시 |

**UI 구조:**
```
📡
인터넷 연결이 끊어졌습니다

Wi-Fi 또는 데이터 연결을 확인해주세요.

[다시 시도]
```

**동작:**
- `window.addEventListener('online', ...)` 로 자동 복구
- 오프라인 중 사용자 액션은 로컬 큐에 보관 (재연결 시 자동 재실행)

| 위치 | 값 |
|---|---|
| **화면** | S-DASH |
| **조건** | 사용자가 거래소 0개 연동 |

**UI 구조:**
```

     🔗
     
     첫 거래소를 연결해 주세요
     
     업비트 API 키를 등록하면 자동매매를 시작할 수 있습니다.
     자산은 거래소에 안전하게 그대로 보관됩니다. ← 비수탁 강조
     
     [거래소 연결하기 →]
     
     [가이드 보기]
```

## ⚪ 3. Loading States (로딩 상태)

### LO-PAGE · 페이지 초기 로드

| 화면 | 로딩 UI |
|---|---|
| S-DASH | 위젯 4개 스켈레톤 |
| S-STRAT | 좌측 채팅 + 우측 블록 스켈레톤 |
| S-BACKTEST | KPI 카드 4개 + 차트 스켈레톤 |
| 기타 | 전체 스켈레톤 |

**규칙:**
- 스켈레톤은 실제 UI와 동일한 위치/크기 (jump 방지)
- shimmer 애니메이션 (1.5초 loop)
- 3초 초과 시 "잠시만 기다려주세요..." 텍스트 추가

---

### LO-WIDGET · 위젯 개별 갱신

**규칙:**
- 위젯 내부에 미세한 페이드 (250ms)
- 로딩 중에도 이전 값 유지 (완전 삭제 X)
- 새 값 도착 시 부드러운 전환

---

### LO-BUTTON · 버튼 액션 중

**UI:**
```
[  스피너 처리 중...  ]  ← 비활성화 + 스피너
```

**규칙:**
- 버튼 라벨 유지 (색만 흐리게)
- 좌측에 스피너 추가
- 클릭 재차 방지

---

### LO-GPT · GPT 응답 대기 ⭐ M09 개선

**UI:**
```
[✓] 이전 대화 메시지

┌────────────────────────────────┐
│  ↻ 전략을 생성하고 있어요...     │
│  보통 5초 정도 걸립니다          │
└────────────────────────────────┘
```

**규칙:**
- 스피너 회전 애니메이션 (1s loop)
- 5초 경과 후: "조금만 더 기다려주세요..."
- 30초 초과 시: "응답이 지연됩니다" + 재시도 버튼

---

### LO-BACKTEST · 백테스팅 실행 중

| 시간 | UI |
|---|---|
| 0-10s | 스피너 + "백테스팅 실행 중..." |
| 10-30s | 진행률 바 (approximated) |
| 30s+ | "복잡한 전략입니다. 잠시만 기다려주세요..." |
| 60s 초과 | 타임아웃 → "잠시 후 다시 시도해주세요" |

## 🎯 예외 처리 공통 원칙

1. **어떤 화면이든 3가지 상태를 정의한다**: Loading / Empty / Error
2. **에러는 사용자 언어로 표현한다** (기술 용어 X)
3. **다음 액션을 명확히 제시한다** — 무엇을 할 수 있는지
4. **자동 복구가 가능하면 사용자에게 알리지 않는다** (E-API 예외 화면 참고)
5. **위험한 액션의 실패는 명확히 알린다** (전략 배포 실패 등)
6. **Loading은 이전 값을 유지한다** (empty 상태로 돌아가지 않음)
