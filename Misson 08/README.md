# 🧪 OpenAI 연동형 자동매매 시스템 — UX 테스트 설계 & 실행

> **작성자: 이제혁**
> Mission 08 / UX Testing Report
> Test period: 2025-06-22 ~ 2025-06-26

Mission 06에서 작성한 와이어프레임을 대상으로, **3명의 실제 사용자**에게 UX 테스트를 진행하고 행동 데이터를 수집한 보고서입니다.

## 📋 태스크 시나리오

각 태스크는 **구체적인 시작점과 성공 조건**을 명시했습니다. 모호한 표현 없이 측정 가능한 형태로 작성했습니다.

### T1. 회원가입부터 거래소 연동까지

| 항목 | 내용 |
|---|---|
| **시작점** | 랜딩 페이지 (W1) |
| **목표** | 업비트 API 키를 등록하여 거래소 연동을 완료한다 |
| **성공 조건** | "권한 검증 완료" 화면 도달 |
| **시간 제한** | 3분 |
| **사전 안내** | "랜딩 페이지에서 시작해서, 가입 후 업비트와 연동해주세요. API 키는 미리 발급된 더미 값을 제공합니다." |

### T2. GPT로 첫 전략 만들기

| 항목 | 내용 |
|---|---|
| **시작점** | 거래소 연동 직후 화면 |
| **목표** | "비트코인 가격이 5% 떨어지면 분할 매수하는 전략"을 만든다 |
| **성공 조건** | GPT 채팅창에 위 요청을 입력하고 응답된 블록을 확인 |
| **시간 제한** | 3분 |
| **사전 안내** | "AI에게 직접 말로 전략을 부탁해보세요. 코딩은 필요 없습니다." |

### T3. 자본 비율을 30%에서 20%로 변경

| 항목 | 내용 |
|---|---|
| **시작점** | GPT가 생성한 전략 (자본 비율 30%) |
| **목표** | 액션 블록의 자본 비율을 20%로 수정한다 |
| **성공 조건** | 자본 비율 표시가 "20%"로 변경됨 |
| **시간 제한** | 2분 |
| **사전 안내** | "AI 추천에서 자본 30%를 20%로 줄이고 싶습니다. 어떻게 하시겠어요?" |

### T4. 백테스팅 결과 보고 배포 결정

| 항목 | 내용 |
|---|---|
| **시작점** | 백테스팅 결과 화면 (W4) — 수익률 +24.7%, MDD -8.2% |
| **목표** | 결과를 보고 "배포할지/수정할지/포기할지" 결정 + 그 이유 |
| **성공 조건** | 명확한 의사결정 + 결정에 사용한 지표 언급 |
| **시간 제한** | 2분 |
| **사전 안내** | "이 백테스팅 결과를 보고, 실제 거래에 배포하시겠어요?" |

### T5. API 오류 발생 시 어떻게 행동할지 (예외 화면 인지)

| 항목 | 내용 |
|---|---|
| **시작점** | API 일시 오류 화면 (W6) |
| **목표** | 이 화면이 무엇을 의미하는지, 사용자가 무엇을 해야 하는지 답한다 |
| **성공 조건** | "사용자 개입이 필요한가?" 질문에 "필요 없음" 응답 |
| **시간 제한** | 1분 |
| **사전 안내** | "회사 수업 중 핸드폰을 보니 이 화면이 떠있습니다. 어떻게 하시겠어요?" |

### T6. 내 자산은 어디에 보관되나요? (비수탁성 인지) ★

## 🔍 관찰된 사용자 행동 (정성 데이터)

> **감상평이 아닌 구체적 행동만** 기록했습니다.

### W3 GPT 전략 화면 (가장 많은 마찰)

#### P1의 행동
- 채팅창에 텍스트 입력 → 전송 버튼 클릭
- **5초 동안 화면을 응시 (응답 대기)** → 마우스를 채팅창 상단으로 다시 가져감
- 우측 블록 편집기를 6초간 응시한 후 첫 블록(트리거) 클릭
- **블록 클릭 후 어떤 화면이 뜨는지 알 수 없어** 다시 채팅창으로 시선 이동 (혼란 행동)
- 발화: *"이거 누르면 뭐가 뜨는 거예요?"*

#### P2의 행동
- GPT 채팅에 자연어 입력 후 **Cmd+Enter 시도** (단축키 메모리 패턴)
- 단축키 작동 안 함 → 전송 버튼 마우스 클릭
- 자본 비율 변경 시도 시 **트리거 블록을 먼저 클릭** (잘못된 블록 선택)
- 5초 후 액션 블록으로 이동하여 클릭

#### P3의 행동 (가장 큰 어려움)
- 채팅창 입력 시 글자 수 모름 → **자연스럽지 않게 매우 짧게 작성** ("BTC 사줘")
- GPT 응답이 너무 길어서 **두 번 다시 읽음** (총 12초)
- 블록 편집기 영역을 **"이게 뭔가요?"라고 발화** 후 무시
- T3에서 자본 비율 변경 시도 → 액션 블록 외 영역 3회 오클릭 → 시간 초과로 **태스크 실패**

### W2 대시보드 — T6 자산 위치 질문

#### P1의 행동
- 화면을 위에서 아래로 천천히 스캔 (8초)
- 자산 위젯 카드를 클릭 (반응 없음) → 사이드바 "거래소" 메뉴 클릭
- "업비트 (비수탁)" 텍스트 발견 후 정답
- 발화: *"이거 클릭해서 들어가야 알 수 있네요"*

#### P2의 행동
- **자산 위젯의 ₩12,450,000 숫자만 응시** (8초)
- "총 자산"이라는 글자만 인식 → 답: *"여기 플랫폼 내부에요"*
- 정답 공개 시 발화: *"아, 거래소에 있어요? 그럼 왜 여기 숫자가 뜨는 거예요?"*
- **태스크 실패 + 핵심 가치 인지 실패**

#### P3의 행동
- 30초간 화면 전체 스캔, 클릭 1회도 안 함
- 답: *"앱에 보관되는 거 아닌가요?"*
- 정답 공개 후: *"아 그게 더 좋은 건데, 왜 어디에도 안 써있어요?"*

### W6 예외 화면 — T5 오류 대응 인지

#### 모든 참가자 공통
- **즉시 4-step 트래커로 시선 이동** (1~2초)
- "자동 재연결 성공" 체크 아이콘 확인 후 5초 내 답
- P1: *"이거 그냥 두면 알아서 해결되는 거네요?"*
- P2: *"수업 중에 이거 떴으면 안심하고 수업 계속 들을 수 있겠어요"* (페르소나 박서연 시나리오 정확 매칭)
- P3: *"버튼을 누를 필요가 없는 거 같아요"*

### W5 API 연동 화면

#### 모든 참가자 공통 행동
- 우측 안내 박스 ("자산은 거래소에 그대로 보관") **3명 모두 정독**
- P2: 안내 박스의 "출금 권한은 절대 활성화하지 마세요" 문구를 손가락으로 짚으며 발화 *"이런 거 좀 강조해줬으면"*
- → Mission 06에서 도출한 "출금 권한 OFF 확인 체크박스" 필요성을 사용자가 직접 검증

## 🎯 핵심 발견사항

### Finding 1 ★ Critical — 비수탁성이 와이어프레임으로 전달되지 않음

**증거:**
- T6 정답률 **33% (1/3)**
- P2, P3 평균 멈춤 시간 **45초**, 클릭 없이 화면만 스캔
- P2 발화: *"여기 플랫폼 내부에요"* (페르소나 핵심 가치 정반대 인지)

**원인:**
- 비수탁성을 강조한 UI 요소가 **W5 API 연동 화면에만** 존재
- 대시보드(W2)에서는 자산 숫자만 표시, 보관 위치는 사이드바 깊이 진입해야 확인 가능

**개선안 (P0):**
- 자산 위젯에 *"업비트 (비수탁)"* 라벨 명시
- 대시보드 상단 영구 표시: *"🔒 자산은 거래소에 안전하게 보관됩니다"*
- 온보딩에서 비수탁성 강조 단계 추가

### Finding 2 — GPT 전략 화면 학습 비용 높음

**증거:**
- W3에서 **누적 멈춤 시간 52초** (전체 화면 중 최대)
- P1 발화: *"이거 누르면 뭐가 뜨는 거예요?"*
- P3: 블록 편집기 영역을 *"이게 뭔가요?"* 후 무시
- T3 완료율 **66%** (P3 실패)

**원인:**
- 블록 클릭 시 무엇이 일어날지 시각적 단서 부족
- GPT 응답 대기 중 로딩 인디케이터 부재 → 5초 동안 사용자 불안

**개선안 (P0):**
- 블록 hover 시 *"클릭하여 편집"* 툴팁 표시
- GPT 응답 대기 중 스피너 + 진행 텍스트 (*"전략을 생성하고 있어요..."*)
- 블록 편집 진입 후 모달 디자인 필요 (와이어프레임에 미반영)

### Finding 3 — W6 예외 화면 디자인은 매우 성공적

**증거:**
